📋 PRD – MVP 核心功能 (个人整理助手)

功能名称：AI 个人整理副驾 · 智能标签 + 推荐文件夹结构

✨ 用户故事示例

功能模块描述技术依赖
🏷️ 自动打标签对文档/图片/PDF语义理解，生成3~5个标签，用户可编辑本地NPU推理 + 轻量云端fallback(小众格式)
📁 智能文件夹推荐基于标签聚类+时间序列推荐移动路径本地聚类算法
📅 会议上下文感知同步日历+可选麦克风，会前自动提取相关笔记置顶(纯本地)需用户授权

📊 成功指标(Beta验证)

  • 整理时间周下降 ≥30%

  • 用户修改/删除标签比例＜20%

🔄 基于反馈迭代

  • ✅ 一键“撤销错误分类”，保留最近10次操作
  • ✅ 增加“排除文件夹”设置（用户希望忽略游戏存档/临时目录）

⚙️ 技术实现思考 | 轻量·隐私·场景聚焦

🎯 模型选型核心思路： 不为全能，只为整理。不采用通用大模型，而是选择轻量级、任务专用的小模型，只做三件事：识别文件类型、提取主题关键词、生成候选标签。所有推理默认在NPU上运行，保证离线可用且隐私安全。

🧩 轻量化策略

  • INT8量化 + 结构化剪枝，体积减小60%以上
  • 动态加载子模型（遇到PDF才加载布局模块）
  • 与NPU团队协作替换激活函数（如ReLU）适配硬件加速
  • 常驻内存 <80MB，后台CPU占用 <3%

☁️ 本地 + 轻量云 fallback

  • 小众文件（设计源文件、专业格式）本地置信度低时，仅上传文件名哈希到云端轻量分类器
  • 云端返回建议标签，本地缓存规则，下次直接本地处理
  • 隐私底线：云端绝不接收文件内容，用户可一键关闭云端fallback

✂️ 能力裁剪：只做整理所需

  • ❌ 不做情感分析、实体关系抽取、摘要生成
  • ❌ 不识别图片中的物体/人脸（除非文件名/OCR文字明确提示）
  • ✅ 只回答：文件类型？核心主题？适合打哪些标签？

🔄 隐私保护的反馈闭环

  • 用户每次修改标签本地记录（不离设备）
  • 可选“帮助改进”：上传脱敏后的“文件类型+最终标签”聚合数据
  • 云端定期微调模型，通过系统更新推送，用户可回滚

📌 一句话总结：用最轻量的模型、最少的计算、最低的隐私风险，只解决文件整理这一个垂直问题。

Qual+Quant

⚙️ Tech Strategy: Lightweight · Private · Task-Focused

🎯 Model selection philosophy: Do one thing well – file organizing. Use small, task-specific models instead of general-purpose LLMs. All inference runs on NPU by default for offline privacy.

🧩 Lightweight techniques

  • INT8 quantization + structured pruning → 60% size reduction
  • Dynamic loading (PDF layout model only when needed)
  • Activation function tuning for NPU acceleration (e.g., ReLU)
  • Resident memory <80MB, CPU <3% in background

☁️ Local + lightweight cloud fallback

  • For niche files, only filename hash sent to cloud classifier (no content)
  • Cloud returns suggested tags; local caches rule for next time
  • Privacy: user can disable cloud fallback completely

✂️ Capability pruning

  • ❌ No sentiment analysis, relation extraction, summarization
  • ❌ No object/face recognition in images
  • ✅ Only answers: file type? core topic? candidate tags?

🔄 Privacy-safe feedback loop

  • Local logging of user tag edits (stays on device)
  • Optional “Help improve”: upload anonymized (file type + final tags) aggregates
  • Quarterly model updates via system update; user can rollback

📌 Bottom line: minimal computation, maximal privacy, solving exactly one vertical problem.
